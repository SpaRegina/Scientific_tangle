# Scientific_tangle

`Scientific_tangle` — конкурсный MVP поисково-аналитической системы для R&D в горно-металлургической отрасли. Проект связывает документы, материалы, процессы, свойства, эксперименты, установки и выводы в граф знаний и отвечает на вопросы на естественном языке.

Важно: в этом репозитории **нет базы знаний** и **нет исходного корпуса документов**. Репозиторий содержит только код, статику и файлы запуска. Базу `data/kg.db` нужно собирать локально из вашего набора источников.

## Что умеет проект

- Загружать и разбирать документы `PDF`, `DOCX`, `DOCM`, `PPTX`, `TXT`, `XLSX`
- Извлекать сущности предметной области на русском и английском языках
- Строить граф знаний на `SQLite + FTS5`
- Выполнять гибридный поиск по графу и полнотекстовому индексу
- Отвечать на вопросы в стиле:
  - что уже делали по материалу X при режиме Y
  - какой был эффект на свойство Z
  - какие публикации и эксперименты связаны с темой
- Показывать источники, связанные сущности, параметры и пробелы в данных
- Экспортировать ответ в `JSON-LD`

## Состав репозитория

- `app/` — FastAPI-приложение и веб-интерфейс
- `kg/` — построение графа, извлечение сущностей, поиск и работа с БД
- `requirements.txt` — зависимости Python
- `run.bat`, `run.sh` — быстрый запуск веб-приложения
- `УСТАНОВИТЬ_И_ЗАПУСТИТЬ.bat` — удобный запуск для Windows
- `.env.example` — шаблон переменных окружения

## Требования

- Python 3.11+ или 3.12
- `pip`
- Локальный набор исходных документов, из которого вы будете строить базу знаний

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и при необходимости укажите:

- `KG_SRC` — путь к папке с вашими документами
- `KG_DB` — путь к локальной базе знаний
- `ROUTERAI_ENABLE=1` и `ROUTERAI_API_KEY=...` — только если хотите включить внешний LLM

По умолчанию проект может работать в rule-based режиме без внешнего LLM.

### 3. Сборка базы знаний

Пример:

```bash
python -m kg.build --src "D:\path\to\your\sources" --db data/kg.db --limit 200
```

Параметры:

- `--src` — папка с документами
- `--db` — путь, куда будет сохранена локальная база знаний
- `--limit` — ограничение числа документов для быстрого демо

### 4. Запуск приложения

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

После запуска откройте:

```text
http://localhost:8000
```

## Переменные окружения

Поддерживаются:

- `KG_SRC` — путь к папке с исходными документами
- `KG_DB` — путь к файлу базы знаний
- `ROUTERAI_ENABLE` — включение внешнего LLM
- `ROUTERAI_API_KEY` — API-ключ внешнего LLM
- `ROUTERAI_MODEL` — имя модели
- `ROUTERAI_BASE_URL` — базовый URL OpenAI-совместимого API

Никакие ключи в этот репозиторий не включены.

## API

- `GET /api/ask?q=...` — ответ, источники, граф, параметры, пробелы, эксперты
- `GET /api/node?canon=...` — карточка сущности, связи, документы, параметры, история фактов
- `GET /api/gaps?type1=Material&type2=Process` — карта пробелов
- `GET /api/graph?terms=a,b` — подграф по терминам
- `GET /api/export?q=...` — экспорт ответа в `JSON-LD`
- `GET /api/stats` — статистика базы
- `GET /api/dashboard` — дашборд руководителя

## Архитектура

```text
документы -> ingest -> extract -> graphdb(SQLite + FTS5) -> search -> FastAPI + SPA
```

Ключевые модули:

- `kg/ingest.py`
- `kg/extract.py`
- `kg/ontology.py`
- `kg/build.py`
- `kg/search.py`
- `kg/graphdb.py`
- `app/main.py`
